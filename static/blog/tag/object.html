yuvalg/blog object 2013-08-13T16:58:22+00:00 https://yuvalg.com/tag/object.html Yuval Greenfield Python Programmers Writing Javascript Beware 2009-11-09T07:38:33+00:00 https://yuvalg.com/2009/11/09/python-programmers-writing-javascript-beware Call me silly for not reading the docs, but this bug was very hard to find. Dictionaries in python are NOT the same thing in javascript. The curly braces mean it's an &quot;object literal&quot; and the keys don't need the quotation marks. For example: var x = { NAME: 123, Something: &quot;ok&quot;} Is the same as the following code: var x = {} x[&#39;NAME&#39;] = 123 x[&#39;Something&#39;] = &quot;ok&quot; Which means that if you had a variable named &quot;NAME&quot; and you expected it's contents to be the key, then you're going to have a royally hard to find bug (especially when developing for an already obfuscated framework such as Google Wave). This is even worse because JSON causes you to believe the quotation marks are needed. If there's a place to file a JS enhancement proposal, I would, but I doubt this inconsistency would annoy anyone but a pythonista.
